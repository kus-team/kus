## Проект: KUS — Kill Unlawful Schemes

- **Название:** **KUS** = *Kill Unlawful Schemes* (имя совпадает с именем директории `D:\KUS`).
- **Прежние имена:** HalolMap (по-узбекски *halol* = «честный»), CorruptScan — оба отвергнуты в пользу KUS.
- **Суть:** система автоматического выявления коррупционных аномалий в госзакупках Узбекистана через анализ открытых данных.
- **Дизайн:** тёмная "control room"-тема, заимствована визуальная база у проекта SauronEye (только CSS-стили, без OSINT-логики).

### Чем НЕ является

- Не сайт для жалоб (слабо).
- Не «AI борется с коррупцией» без реальной логики (жюри видит насквозь).
- Не крутая идея без реализации (проигрыш).

### Чем ЯВЛЯЕТСЯ

- Реальные данные с `data.egov.uz`.
- Математический анализ аномалий.
- Визуализация рисков.
- Работающее демо.

**Вдохновение:** сингапурский подход — data-driven контроль, прозрачность, минимизация человеческого фактора.

## Данные

- **Источник:** https://data.egov.uz — Портал открытых данных Республики Узбекистан.

### API портала (нашли реверсом Nuxt-чанков)

| Назначение | Метод / URL |
|---|---|
| Поиск датасетов | `GET /apiClient/main/gettable?limit=&offset=&text=&orgId=&sphereId=&regionId=` |
| Метаданные датасета (поля, организация) | `GET /apiClient/Main/GetById?id=<structId>` |
| Данные датасета (JSON) | `GET /apiData/MainData/GetByFile?id=<structId>&fileType=1&tableType=2` |
| То же в CSV / XLS / XML / RDF | `fileType`=4 / 3 / 2 / 5 |
| Список сфер | `GET /apiClient/Main/GetSphereList` |

### Почему исходный датасет `61023b292a2e256d868e8285` не подходит

«All contests and tenders of state bodies» (Минфин, код 1-002-0008) — это **агрегированная статистика** с полями
`Xaridturi / Bitimsoni / Bitimsummasi / Foizda`. Последнее обновление — **2023-07-31**. Для индекса риска по конкретным
тендерам не подходит.

### Что вместо него — сырые датасеты тендеров

На портале ~130+ датасетов вида «Information on public procurement carried out by …» — по одному на госорган.
Они содержат **строки отдельных закупок** с нужными нам полями. Топ-кандидаты по объёму и актуальности:

| structId | Код | Организация | Обновлён | Записей (sample) |
|---|---|---|---|---|
| `613eeda614665dbb8ec80453` | 1-001-0008 | Минэкономики и финансов | 2026-03-03 | 98 |
| `6142ef46ba3615f6f07bca0f` | — | (по теме госзакупок) | — | 19 |
| `64dc981eb04a41cb2e29d57b` | — | Минэкономики и финансов | — | 294 |
| `6225c27ed31e97c0521ec8a1` | 6-009-0027 | Хокимияты (аукционы земли) | — | 302 |
| `61137447db32b99538e086fc` | 4-019-0010 | Алмалыкский ГМК | 2026-04-23 | 1982 |

Всего по поиску найдено **157 уникальных** датасетов, связанных с закупками/тендерами (скрипт `scripts/explore_datasets.py`).

### Схема сырого датасета (пример — 1-001-0008)

```json
{
  "BuyurtmachiSTIRraqami": "201122919",            // ИНН заказчика
  "Predmetimahsulotishxizmat": "Asbob-uskunalarni…", // предмет закупки
  "Kategoriyasi": "…",                             // категория
  "Miqdoriolchovbirligi": "1",                     // количество / ед. измерения
  "Lotraqami": "251191180227671",                  // № лота
  "Moliyalashtirishmanbai": "Budjet mablag'lari",  // источник финансирования
  "YetkazibberuvchinomivaSTIRraqami": "306328693", // ИНН победителя/поставщика
  "Yetkazibberishmuddatikunishkuniyokisutka": "7", // срок поставки, дней
  "Togridantogrixaridamalgaoshirishasosi": "Прямые договора (ЗРУ-684 Ст-71 п.2)", // основание прямой закупки
  "Shartnomaraqami": "ET-Sh-025-551",              // № контракта
  "Shartnomasanasi": "2025-12-22",                 // дата контракта
  "Shartnomaqiymati": "19873"                      // сумма (в тыс. сум)
}
```

Схема от датасета к датасету немного плывёт (имена полей местами без «ning», даты в разных форматах). Поэтому нужен
нормализатор на этапе загрузки.

### Как поля ложатся на 3 красных флага индекса риска

- **Монополизация:** group by `(BuyurtmachiSTIRraqami, YetkazibberuvchiSTIR)` → если пара встречается > 5 раз.
- **Отсутствие конкуренции:** `Togridantogrixaridamalgaoshirishasosi` содержит «Прямые договора / To‘g‘ridan to‘g‘ri» — +30.
- **Завышенная цена:** средняя `Shartnomaqiymati` по `Kategoriyasi` → если цена выше на 30 %+ — +40.

### Пробелы, которые надо закрыть

- Названия юрлиц по ИНН (в данных только ИНН победителя; человеко-читаемое имя — не всегда). Нужен отдельный справочник ИНН → название.
- Число участников тендера (`participants_count` в схеме БД) прямо в этих датасетах **не указано** — пересматриваем: вместо него используем «прямая закупка без конкурса» (флаг по `Togridantogri…`).
- Единицы `Shartnomaqiymati` — где-то сум, где-то «тыс. сум». Нужно нормализовать по датасету.

## Алгоритм анализа — ключевая фишка

### Индекс риска коррупции (0–100)

Три красных флага, каждый даёт баллы; итог суммируется и клипуется до 100.

| Красный флаг | Баллы | Условие | Что означает |
|---|---|---|---|
| Монополизация | **+30** | Один победитель (ИНН) выиграл у одного заказчика (ИНН) **более 5 раз** | «Своя компания» |
| Завышенная цена | **+40** | `Shartnomaqiymati` **выше средней по `Kategoriyasi` на 30 %+** | Откат / взятка |
| Отсутствие конкуренции | **+30** | `Togridantogrixaridamalgaoshirishasosi` содержит «Прямые / To‘g‘ridan» (прямая закупка без конкурса) | Предварительная договорённость |

### Пороги

- **0 – 30** — зелёный, норм
- **30 – 70** — жёлтый, подозрительно
- **70 – 100** — красный, высокий риск

### Пример

- Тендер 100 млн сум при средней по категории 70 млн → **+40**
- Победитель выиграл у заказчика 8 раз → **+30**
- Один участник → **+30**
- **Итого: 100 → красный флаг.**

## Архитектура

### База данных (PostgreSQL)

```sql
-- Основная таблица тендеров (обновлено под реальную схему egov.uz)
tenders:
  id                  PRIMARY KEY
  source_dataset      TEXT           -- structId исходного датасета
  lot_id              TEXT           -- Lotraqami
  contract_id         TEXT           -- Shartnomaraqami
  title               TEXT           -- Predmetimahsulotishxizmat
  customer_tin        TEXT           -- BuyurtmachiSTIRraqami (ИНН заказчика)
  customer_name       TEXT           -- из справочника ИНН→name
  winner_tin          TEXT           -- YetkazibberuvchiSTIR
  winner_name         TEXT           -- из справочника или из поля, где оно есть
  amount              NUMERIC        -- Shartnomaqiymati, нормализовано к сумам
  currency            TEXT           -- Valyuta (UZS по умолчанию)
  date                DATE           -- Shartnomasanasi
  category            TEXT           -- Kategoriyasi
  funding_source      TEXT           -- Moliyalashtirishmanbai
  purchase_method     TEXT           -- XaridTuri / Togridantogri...
  is_direct_purchase  BOOL           -- true если прямая закупка без конкурса
  risk_score          INT            -- 0–100
  risk_flags          JSONB          -- {"monopoly":true,"overpriced":true,"no_compete":true}
  created_at          TIMESTAMPTZ DEFAULT now()
  UNIQUE (source_dataset, lot_id, contract_id)

-- Кэш расчётов для быстрого скоринга
analytics_cache:
  category            TEXT PRIMARY KEY
  avg_price           NUMERIC
  total_count         INT
  updated_at          TIMESTAMPTZ
```

### Backend API (FastAPI)

| Метод | Путь | Назначение |
|---|---|---|
| GET  | `/api/tenders` | Список тендеров с пагинацией и фильтрами |
| GET  | `/api/tenders/{id}` | Детали тендера |
| GET  | `/api/tenders/suspicious` | Тендеры с `risk_score > 70` |
| GET  | `/api/analytics/stats` | Общая статистика (счётчики, суммы) |
| GET  | `/api/analytics/risk-map` | Данные для карты рисков |
| POST | `/api/check-tender` | Проверка конкретного тендера (killer feature) |
| GET  | `/api/company/{name}` | Анализ компании (все её тендеры, паттерны) |

### Frontend — страницы

1. **Dashboard** — карточки (всего тендеров, подозрительных, общая сумма), график динамики по месяцам, топ-5 подозрительных компаний.
2. **Таблица тендеров** — поиск, фильтры, цветовой индикатор риска, сортировка.
3. **Проверка тендера** *(killer feature)* — ввод ID/названия, риск-скор, список красных флагов, сравнение с аналогами.
4. **Анализ компании** — поиск по имени, все выигранные тендеры, график сумм, паттерны (у каких заказчиков чаще выигрывает).
5. **Карта рисков** *(опционально)* — если есть геоданные, точки по регионам с цветовой заливкой.

## Стратегия победы

1. **Локальный контекст** — не абстракция, а конкретно Узбекистан и реальные данные `data.egov.uz`.
2. **Data-driven** — математика и алгоритмы, а не «AI» для красоты.
3. **Работающее демо** — живой прототип, который можно потрогать на презентации.
4. **Визуализация** — карта, графики, dashboard. Жюри любит визуал.
5. **Конкретная польза** — любой человек может проверить тендер; журналист найдёт аномалии; чиновник улучшит прозрачность.

**Питч на презентации:**
> «В Узбекистане проблема не в отсутствии данных о госзакупках — они открыты на data.egov.uz. Проблема в том, что их никто не анализирует системно. Мы создали платформу, которая превращает тысячи тендеров в понятные сигналы о коррупционных рисках. За 10 дней мы проанализировали [X] тендеров и выявили [Y] подозрительных паттернов.»
